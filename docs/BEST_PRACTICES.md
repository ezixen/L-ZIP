# Best Practices Guide

This page is the canonical docs entry for best-practice usage and points to the full guide maintained at repository root.

## Primary Guide
- ../BEST_PRACTICES.md

## Practical Rules
- Keep prompts structured and explicit.
- Use clear operator ordering.
- Prefer specific objectives over vague shorthand.
- Add minimal plain-English clarification when nuance is critical.
- Validate round-trip expansion for high-risk workflows.

## Domain Tips
- Code tasks: specify language/framework and output artifacts.
- Analysis tasks: provide explicit context and evaluation criteria.
- Writing tasks: provide audience, tone, and length constraints.

## Validation Checklist
- Intent preserved after compression
- Token reduction is meaningful for your use case
- Output format requirements are explicit
- No ambiguous operator values
